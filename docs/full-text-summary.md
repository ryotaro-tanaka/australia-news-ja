# 🎯 プロジェクト概要
英語ニュース（主に ABC News）を日本語で読みたいユーザー向けに、  
**タイトル・短い要約・長い要約（本文風）** を自動生成して表示するニュースサイトを構築する。

ユーザーの大半はスマホ利用のため、**スマホ特化 UI** を最優先とする。

---

# 🧩 現状の課題
- 現在は「タイトル＋短い要約」だけを日本語で表示し、クリックすると英語の元記事へ飛ぶ。
- その結果、ユーザーがっかり → 離脱率が非常に高い。
- 本文（日本語）が必要。

---

# 🛠️ 今回決定した仕様

## 1. **モーダルは使わない**
スマホ UX を考慮し、モーダルは不採用。

理由：
- 閉じる操作が煩わしい
- 二重スクロール問題
- URL が変わらずシェアしづらい
- 戻るボタンで閉じないケースがある

---

## 2. **記事は縦に伸びる構造にする（スマホ最適）**

### 初期表示（一覧）
- タイトル（日本語）
- 短い要約（日本語）

### クリック後（詳細ページ）
- タイトル（日本語）
- **長い要約（本文風の再構成記事）**
- 最後に「元記事（英語）へのリンク」

※ 長い要約は「翻訳」ではなく「再構成記事」として生成する  
（著作権対策・読みやすさ向上）

---

## 3. **長い要約（本文風）の構成方針**
- **自然な文章**: 日本語ニュースサイトのような自然な語り口で再構成する。
- **禁止事項**: 
  - 「【結論】」「【背景】」などの見出しや箇条書きは一切使用しない。
  - セクション名を付けない。
  - AI特有のテンプレート構造や不自然な翻訳調を避ける。
- **構成**: 3〜6段落程度。第1段落で結論を述べ、以降で背景・詳細・コメント・展望を織り交ぜる。
- **スマホ最適化**: 段落ごとに適切な改行を入れ、縦長画面での可読性を高める。

---

## 4. **AI モデル（Workers AI / Llama 3 8B）で生成**
- 英語記事を取得し、日本語ニュースとして再構成する。
- **KVストレージ**: 既存の `ja:URL` キーの値を拡張し、`{"title": "...", "line": "...", "bodyJa": "..."}` 形式で保存する。
*注: 本機能のリリースに合わせて既存の KV データは一度クリアする。*
- **生成タイミング**: 記事の一覧取得・翻訳プロセス時に一括で生成する（オンデマンド生成は行わない）。

---

## 5. **UI / UX の方針**
- スマホでの読みやすさを最優先
- 記事詳細ページはページ遷移として実装
- 戻るボタンで自然に一覧へ戻れる
- URL は `/news/{id}` の形式（ID は URL の SHA-256 ハッシュ）

---

# 📌 実装内容

## データソースの抽出ロジック
- **短い要約 (line)**: RSS フィードの `<description>` タグをベースに翻訳。一覧画面で使用。
- **長い要約 (bodyJa)**:
    1.  **フェッチ**: RSS の `<link>` タグにある ABC News の記事 URL を `fetch` で取得。
    2.  **抽出 (HTMLRewriter)**: Cloudflare の `HTMLRewriter` を使用し、ストリーム解析で本文を抽出。
        - ターゲット: 本文が含まれる `<p>` タグ。
        - 最適化: 必要に応じて記事メインエリア（例: `[data-component="Text"]` 等）に絞り込み、広告やサイドバーのノイズを除去。
    3.  **再構成**: 抽出した英語全文テキストを AI（Llama 3 8B）に渡し、指定のスタイルで日本語記事として再構成。

## API設計
- **エンドポイント**: `functions/api/news.ts`
- **一括処理**: RSS フェッチ時に各記事に対して上記抽出〜AI生成プロセスを実行し、KV を更新する。
- **ID設計**: `generateId(url)` を用いて URL の SHA-256 ハッシュを生成し、一意の ID とする。

## UIコンポーネント構成
- **一覧**: `NewsList.tsx` -> `NewsCard.tsx` (内部リンクに変更)
- **詳細**: `NewsDetail.tsx` (新規作成)

# 📝 実装プラン

## ステップ1: フロントエンドの基盤整備
- **ライブラリ**: `react-router-dom` を導入。
- **ルーティング**: `App.tsx` で `/` と `/news/:id` を設定。
- **型定義**: `src/types/news.ts` に `id: string` と `bodyJa: string` を追加。

## ステップ2: バックエンドの拡張
- **ID生成**: URL からハッシュ ID を生成するロジックの実装。
- **AIプロンプト**: 「自然な日本語ニュース」生成のためのプロンプト調整。
- **KV保存**: `bodyJa` を含む拡張 JSON 形式での保存処理。

## ステップ3: 詳細画面の実装
- **コンポーネント**: `NewsDetail.tsx` の作成。
- **スタイル**: モバイル向けに段落間隔やフォントサイズを最適化した CSS。

## 検証項目
- `/news/{id}` で生成された日本語本文が表示されること。
- AI生成物に「【結論】」などの不要な見出しが含まれていないこと。
- スマホブラウザの戻るボタンで正常に動作すること。

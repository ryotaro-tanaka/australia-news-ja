# 🎯 プロジェクト概要
英語ニュース（主に ABC News）を日本語で読みたいユーザー向けに、  
**タイトル・短い要約・長い要約（本文風）** を自動生成して表示するニュースサイトを構築する。

ユーザーの大半はスマホ利用のため、**スマホ特化 UI** を最優先とする。

---

# 🧩 現状の課題
- 現在は「タイトル＋短い要約」だけを日本語で表示し、クリックすると英語の元記事へ飛ぶ。
- その結果、ユーザーがっかり → 離脱率が非常に高い。
- 本文（日本語）が必要。

---

# 🛠️ 今回決定した仕様

## 1. **モーダルは使わない**
スマホ UX を考慮し、モーダルは不採用。

理由：
- 閉じる操作が煩わしい
- 二重スクロール問題
- URL が変わらずシェアしづらい
- 戻るボタンで閉じないケースがある

---

## 2. **記事は縦に伸びる構造にする（スマホ最適）**

### 初期表示（一覧）
- タイトル（日本語）
- 短い要約（日本語）

### クリック後（詳細ページ）
- タイトル（日本語）
- **長い要約（本文風の再構成記事）**
- 最後に「元記事（英語）へのリンク」

※ 長い要約は「翻訳」ではなく「再構成記事」として生成する  
（著作権対策・読みやすさ向上）

---

## 3. **長い要約（本文風）の構造テンプレート**
以下の構造で統一する：

1. **何が起きたか（結論）**  
2. **背景**  
3. **詳細**  
4. **関係者のコメント**  
5. **今後どうなるか**  
6. **元記事リンク**

※ スマホで読みやすいように段落を短くし、適度に改行を入れる。

---

## 4. **AI モデル（Workers AI / Llama 3 8B）で生成**
- 英語記事を取得
- 上記テンプレートに沿って日本語で再構成
- KV にキャッシュして毎回生成しないようにする

---

## 5. **UI / UX の方針**
- スマホでの読みやすさを最優先
- 記事詳細ページは「モーダル風の見た目」だが実際はページ遷移
- 戻るボタンで自然に一覧へ戻れる
- URL は `/news/{id}` の形式

# 📝 実装プラン

## ステップ1: フロントエンドのルーティング導入と詳細画面の雛形
- **ライブラリ追加**: `react-router-dom` を導入。
- **ルーティング設定**: `App.tsx` を修正し、`/` (一覧) と `/news/:id` (詳細) のパスを処理可能にする。
- **型定義更新**: `src/types/news.ts` に `fullSummaryJa?: string` プロパティを追加。
- **詳細コンポーネント**: `src/components/news/NewsDetail.tsx` を作成し、詳細ページのレイアウトと「戻る」ボタンを実装。

## ステップ2: バックエンド詳細 API の実装
- **エンドポイント**: `functions/api/news/detail.ts` (またはクエリパラメータ) を作成。
- **ID設計**: 記事URLのハッシュ値（SHA-256）を ID として使用し、URL から ID を生成して KV のキーとする。
- **データ取得**: ID を元に KV からタイトル・URLを取得し、必要であれば後述の要約ロジックを走らせる。

## ステップ3: 全文取得とAI再構成ロジックの実装
- **本文抽出**: `functions` 内で記事URLから HTML をフェッチし、本文テキストを抽出するロジックを実装。
- **AIプロンプト**: Llama 3 8B を使用し、以下のテンプレートに従って日本語で再構成するプロンプトを作成。
  1. 何が起きたか（結論）
  2. 背景
  3. 詳細
  4. 関係者のコメント
  5. 今後どうなるか
- **キャッシュ**: 生成した長文要約を KV に `full:ja:{id}` のキーで保存（有効期限は既存と同様）。

## ステップ4: UI/UX の洗練とモバイル最適化
- **CSS設計**: 詳細ページ用のスタイルを作成。段落間隔を広めに取り、スマホでの可読性を高める。
- **遷移演出**: 一覧のカードをクリックした際、自然に詳細ページへ遷移するようにする。
- **フォールバック**: AI生成が失敗した場合の「元記事リンク」への誘導を確実に行う。

## 検証項目
- `/news/{id}` に直接アクセスした際に、該当記事の長文要約が表示されること。
- KV にキャッシュが保存され、二回目以降の表示が高速化されること。
- スマホブラウザの「戻る」ボタンで一覧ページに正しく戻れること。

# 🔚 補足
- サイトは Cloudflare Pages + Workers AI + KV を使用。

# React 責任分離モデル (豪州ニュースプロジェクト)

このドキュメントは、プロジェクトが成長してもコードの可読性と保守性を維持するための、Reactコンポーネントの責任分離ルールを定義します。

## ゴール

- UIの振る舞いを予測可能にする。
- 「何を表示するか」と「どのようにデータを取得/更新するか」を分離する。
- ファイルが肥大化する前に、役割ごとに自然に分割できるようにする。

## 基本原則

各機能は以下のレイヤーに分離します：

1. **View Components (表示レイヤー)**
  - 純粋なレンダリングとローカルなインタラクションのみを担当。
  - API呼び出しなどのサイドエフェクトを直接持たない。
  - データやコールバックは props または専用の Context Hook から受け取る。
2. **Feature Containers (コンテナレイヤー)**
  - 画面の特定のエリア（例：`Header`, `NewsList`, `Settings`）を構成するために View を組み合わせる。
  - グローバルな状態（Context）を読み取り、アクションをトリガーする。
3. **State Layer (状態管理レイヤー - Context + useReducer)**
  - 共有される状態と Reducer を保持。
  - アクション（Dispatch）を通じて状態遷移を明示的に行う。
  - ここに大きな JSX は記述しない。
4. **Usecase Layer (非同期/ロジックレイヤー)**
  - API（`/api/news`）の取得、Workers AI の呼び出し等のサイドエフェクトを管理。
  - Reducer アクションをディスパッチする。
  - スタイリングは持たず、ビジネスロジックに集中する。

## 状態管理の使い分け

### 共有状態 (Context)

複数のコンポーネントで必要、またはアプリ全体に影響する状態。
例：
- 取得済みのニュースデータ
- 選択中のカテゴリー / 検索クエリ
- ローディング / エラー状態
- ダークモードの設定

### ローカル状態 (useState)

特定のコンポーネント内だけで完結する一時的な状態。
例：
- フォームの入力内容
- モダールの開閉状態（そのコンポーネント内のみで使う場合）

## コンテキスト境界

- **`newsData`**
  - ニュース項目、翻訳データ、最終更新時間。
  - APIフェッチとキャッシュ管理（KV連携等）のロジックを所有。
- **`appUI`**
  - 画面の表示モード（リスト/グリッド）、フィルター設定、通知（Toast）の状態。

## 命名規則と配置

- **Components (`src/components/`)**
  - `*View`: 純粋な表示用
  - `*Card`: 個別のニュースカード
  - `*List`: 一覧表示
- **Hooks (`src/hooks/`)**
  - `useNewsFlow`: ニュース取得の一連の流れ
  - `useTranslation`: 翻訳に関連するロジック
- **State (`src/state/`)**
  - 各 Context と Reducer

## ファイル配置モデル

- `src/components/ui/`: ボタンやカードなどの再利用可能な最小パーツ
- `src/components/news/`: ニュース表示に特化した View
- `src/state/`: Context / Reducer
- `src/hooks/`: ロジック、API呼び出し、副作用の管理
- `functions/api/`: Cloudflare Pages のバックエンドロジック

## App.tsx の役割

`App.tsx` は最終的に以下のみを担当するようにします：
- Context Provider のマウント
- 画面全体のレイアウト構造（Shell）の定義
- 主要なコンテナの配置

直接の `useEffect` による API 取得や、詳細なスタイリングは避けます。

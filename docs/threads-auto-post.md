# Threads 自動投稿システム

このドキュメントでは、本プロジェクトに実装された Threads へのニュース自動投稿機能の仕様と運用方法について解説します。

## 1. 概要
`npm run post` コマンドを実行することで、自社サイトの API から最新ニュースを取得し、Gemini による要約を経て、画像付きで Threads に投稿します。また、投稿の直後にはサイトへの誘導を含む返信（スレッド）を自動的に作成します。

## 2. 事前準備

動作には以下の環境変数が `.env` ファイルに設定されている必要があります。

*   `THREADS_LONG_LIVED_TOKEN`: Meta for Developers で発行した長期アクセストークン。
*   `gemini` コマンド: ローカル環境に `gemini cli` がインストールされ、パスが通っていること。

## 3. コマンド

| コマンド | 用途 |
| :--- | :--- |
| `npm run post` | **推奨**：最新ニュースを自動で要約・投稿（全自動） |
| `npm run post:raw "本文" "画像URL"` | 任意のテキストと画像URLを手動で投稿（マニュアル） |

## 4. 処理フロー (`npm run post`)

1.  **最新記事の取得**: `https://news-ja.pages.dev/api/news?nocache=1` から最新の1件を取得。
2.  **記事本文の抽出**: 記事URLから HTML をフェッチし、不要なタグを除去してテキストを抽出。
3.  **Gemini による要約**: 抽出したテキストをローカルの `gemini` コマンドに渡し、オーストラリア在住日本人向けの要約（約250文字）を生成。
4.  **メイン投稿の作成**: 要約、記事サムネイル、ハッシュタグ（#オーストラリア）、およびカテゴリタグ（オーストラリア）を Threads に POST。
5.  **API反映待ち**: メイン投稿が API 経由で認識されるまでポーリング（最大10回）を行い、返信が可能な状態になるのを待機。
6.  **自動返信（スレッド化）**: メイン投稿に紐づく形で、サイトURLを含む固定メッセージを投稿。

## 5. 技術的な工夫と制限事項

### 文字数制限への対応
Threads API は1投稿あたり 500 文字の制限があります。本システムでは以下の2段階で対策しています：
*   **プロンプト制御**: Gemini に対し、URL等を含めて 400 文字以内に収めるよう指示。
*   **自動切り詰め**: 投稿直前に 500 文字を超えている場合、スクリプトが自動的に 497 文字でカットし `...` を付与。

### 投稿の安定性
API へのリクエスト直後は Meta 側のデータ同期が間に合わず、返信（スレッド化）に失敗することがあります。これを防ぐため、メイン投稿の ID が自分の投稿一覧に出現するまでチェックする「ポーリング処理」を実装しています。

### 画像の扱い
Threads API の制約により、画像は「インターネットからアクセス可能な公開URL」である必要があります。ローカルのファイルパスを直接投稿することはできません。
